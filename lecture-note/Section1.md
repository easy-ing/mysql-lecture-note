용어 정리

- 뷰 : 특정 쿼리의 결과를 이름으로 저장해서, 마치 테이블을 조회하는 것 처럼 보여주는 가상의 테이블 → 가상으로 저장하기 때문에 주로 일회성으로 정리한다
- B-Tree : mysql에서 가장 기반이 되는 인덱스 구조로 특정 인덱스 키를 기준으로 정렬 하는 방식을 의미한다
- 인덱스 : 데이터를 빠르게 찾기 위해 사용하는 구조이다. 책의 목차와 같은 개념임
- Exclusive Lock (=X Lock, 배타 락) : 하나의 트랜잭션만 특정 데이터에 대해 읽기와 쓰기를 모두 독점하는경우, 혹은 쓰기만 독점하는 동시성 제어 장치이다.
- Anchor : 핵심 적인 엔티티 역할을 수행하는 테이블을 의미한다
- Table : 데이터를 행과 열의 형태로 구조화해 저장하는 기본 단위
- Foregin Key(=외래키) : 테이블간의 관계를 맺고 데이터의 일관성을 유지하는데 있어 가장 기본이 되는 제약조건 역할을 수행
- Transaction : 여러 데이터 연산을 하나의 작업단위로 묶어 모두 성공하거나 모두 실패하는 것 = All or Nothing
- InnoDB : Mysql에서 가장 기본적으로 사용되는 스토리지 엔진
트랜잭션 지원, 외래키 제약, 자동 크래시 복구 기능을 제공한다
